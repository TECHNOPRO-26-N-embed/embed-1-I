### 1-2. 状態遷移（ガジェットの「ふるまい」設計）

> ※ ガジェットが「どんな状態を持つか」を定義します。
> この設計が loop() の骨格になります。

**記入例：**
```
[電源ON / 初期化]
  ↓（初期化完了）
[待機中] ──（センサー検知）──→ [計測中] ──（閾値超え）──→ [警告中]
   ↑                                                            │
   └────────────────────（リセットボタン）────────────────────────┘
```

**↓ 自分の状態遷移を書いてください**
```
[電源ON / 初期化]
  ↓（初期化完了）
[待機中] ──（距離測定開始）──→ [計測中] ──（距離 > 100cm）──→ [無音状態]
   ↑                              │
   │                              ├─（50〜100cm）──→ [弱警告]
   │                              │
   │                              ├─（20〜50cm）──→ [中警告]
   │                              │
   │                              └─（20cm以下）──→ [強警告]
   │                                                    │
   └──────────────（継続測定 / 更新）───────←───────────┘

```

**各状態の意味：**

| 状態名 | この状態のとき、何をしているか | LEDやブザーなど出力の様子 |
|:--|:--|:--|
| （例）待機中 | センサー値を100msごとに監視する | 緑LED：500msで点滅 |
| 初期化中 | センサ・ピン設定を行う | 無音 |
| 待機中   | 測定開始待ち（すぐ計測へ遷移） | 無音 |
| 計測中   | 距離を周期的（約100ms）に測定する | 無音 |
| 無音状態 | 距離が100cm以上のため警告なし | ブザーOFF |
| 弱警告   | 距離50〜100cmを検知 | ブザー：1000Hz・1秒周期で断続音 |
| 中警告   | 距離20〜50cmを検知 | ブザー：1500Hz・0.4秒周期で断続音 |
| 強警告   | 距離20cm以下を検知 | ブザー：2000Hzで連続音 |
| エラー状態 | 測定値異常（0や範囲外）を検知 | ブザーOFF（または前回状態維持） |

> [!TIP]
> すべての状態から「抜け出せるルート」があるか確認してください。
> 「このボタンを押せば必ずリセットに戻る」が保証されていますか？

### 2-2. 機能一覧・関数一覧（役割の定義）

> loop() の中で全部書こうとしていませんか？

### 2-3. タイミング設計（delay vs millis）

> ※ `delay(N)` はN ms間、**すべての処理**が止まります。
> 複数の処理を並行させたい場合は `millis()` を使います。

| 処理       | 必要な周期                 | delay / millis    | 判断の理由                    |
| -------- | --------------------- | ----------------- | ------------------------ |
| 距離測定     | 100ms                 | millis            | センサ値を安定して更新しつつ他処理と並行するため |
| 弱警告（断続音） | 1000ms（ON/OFF切替500ms） | millis            | ブザー制御と距離測定を同時に行うため       |
| 中警告（断続音） | 400ms（ON/OFF切替200ms）  | millis            | 高速な音変化でも処理を止めないため        |
| 強警告（連続音） | 常時                    | millis            | tone出力を維持しながら他処理を継続するため  |
| 無音状態     | 不要                    | millis            | 状態監視を継続しながら即時復帰するため      |

> [!CAUTION]
> `delay()` を使う場合、その間はボタン入力もセンサー読み取りも**すべて止まります**。

## 4. 異常系・エラー処理の方針


| 異常ケース             | 発生条件                | 検知方法                     | 対応方針                                  |
| :------------------ | :---------------------- | :--------------------------- | :----------------------------------------- |
| センサー異常値（0）      | 物体が検知できない／反射なし    | 距離が0cm                     | 危険回避のため短周期警告音へ遷移（前回値は不使用） |
| センサー異常値（過大）     | 測定範囲外の反射            | 距離が200cm超                  | 有効範囲外として短周期警告音へ遷移                 |
| 測定ばらつき（ノイズ）     | 距離が高速に変動            | 前回値との差が大きい           | 中央値/平均化処理で平滑化し、異常連続時は警告音      |
| 閾値付近の揺れ         | 距離が閾値付近を行き来         | 状態が頻繁に変化               | ヒステリシスを固定（例: 50cm境界は50/55で判定）     |
| センサ未接続          | 配線ミス・断線             | 常に0または不定値                | 危険回避のため短周期警告音を維持                  |
| pulseInタイムアウト   | エコー信号が返らない          | pulseInのtimeout到達           | 1回目は再試行、3回連続で短周期警告音へ遷移            |
| ブザー異常（鳴りっぱなし）   | 制御不具合               | noToneが呼ばれない              | 状態遷移時に必ず停止処理実行                      |
| 電源不安定           | USB給電不足             | 値が不安定・再起動                | 短周期警告音へ遷移しつつ測定を再初期化                |
| 処理遅延            | loop処理過負荷           | 更新周期が遅延                  | millisによる周期管理、遅延時はデバッグ出力で検知       |

### Q1: 設計の妥当性確認

> 「この基本設計書をもとに Arduino UNO R3 でコードを書きます。ピン競合・ライブラリ競合・状態遷移の抜け・delay() の問題点はありますか？」
